## b) Tee Kalin ja Metasploitablen välille virtuaaliverkko. Jos säätelet VirtualBoxista
Kali saa yhteyden Internettiin, mutta sen voi laittaa pois päältä
Kalin ja Metasploitablen välillä on host-only network, niin että porttiskannatessa ym. koneet on eristetty intenetistä, mutta ne saavat yhteyden toisiinsa

### Halutaan siis kali ja metasploitablen välille yhteys, mutta ei pääsyä internettiin muulle kuin Kali.

Löysin VirtualBox manuaalin netistä, jossa selitetään virtuaaliverkkojen luomista. [VirtualBox manual chapter 6. Virtual networking](https://www.virtualbox.org/manual/ch06.html)

Host-Only Network, host-kone on yhteydessä virtuaalikoneisiin ja ne voivat olla yhteydessä toisiinsa sekä host-koneeseen.
Manuaalin kohdassa 6.7. Host-Only Networking, selitetään miten se otetaan käyttöön.
- Avataan virtuaalikoneen asetukset ja navigoidaan kohtaan Network.
- Valitaan Adapter ja ruksataan Enable Network Adapter.
- Valitse Attached to: Host-Only Adapter.

Valitsin Kali:lle Host-Only toiseksi verkkoadapteriksi. Sen ensimmäinen verkkoadapteri on oletuksena NAT.

![]()

Asetan sen Metasploitable virtuaalikoneelle ensimmäiseksi ja ainoaksi verkkoadapteriksi, joten se on ainoa verkko minkä se näkee.

![]()

### Pikku miete!
Tehtävänannossa mainitaan host-only network, mutta manuaalissa oli vielä toinenkin sopivan oloinen verkkotyyppi, internal networking.
Se verkko on vain valikoiduille virtuaalikoneille saatavilla, muttei host-koneella pyöriville ohjelmille tai ulkomaailmalle. [VirtualBox manual Chapter 6.6. Internal Networking](https://www.virtualbox.org/manual/ch06.html)

## c) Harjoittelemme omassa virtuaaliverkossa, jossa on Kali ja Metaspoitable. 
Osoita testein, että 1) koneet eivät saa yhteyttä Internetiin 2) Koneet saavat yhteyden toisiinsa.
### Ensin Kalin pitää katkaista NAT verkkoadapterin yhteys, jotta se ei saisi yhteyttä verkkoon.
Työpöydällä on ethernet portin kuvake, josta aukeaa verkkoyheyspaneeli. Sieltä voin katkoa ja yhdistää verkkoyhteyksiä nopeasti.

![Kali Disconnect]()

### Jotta voin testailla verkkoyhteyksiä, minun pitäisi tietää koneiden IP-osoitteet. 
Selvitin sen komennolla:
```
ifconfig
```
![Metasploitable ifconfig]()

![Kali ifconfig]()

### Nyt testaan saako koneet yhteyden internettiin tai toisiinsa.
Käytän ping komentoa testatakseni molemmat yhteydet. Ensimmäinen ping testaa yhteyden internettiin lähettämällä datapaketteja googlen nimipalvelimelle osoitteessa 8.8.8.8
```
ping 8.8.8.8
```
Toinen ping komento testaa yhteyden koneiden välillä.
```
ping 192.168.56.103 -c 4
```
```
-c 4, parametri rajoittaa vain neljään pakettiin.
```

![]()

```
ping 192.168.56.101
```
![]()

### Voin todeta, että kumpikaan ei saa yhteyttä internettiin, mutta löytävät toisensa.
Jos ping 8.8.8.8 saisi paketteja lähetettyä niin koneet olisivat yhteydessä internettiin.

## d) Etsi Metasploitable porttiskannaamalla (nmap -sn). 
Tarkista selaimella, että löysit oikean IP:n - Metasploitablen weppipalvelimen etusivulla lukee Metasploitable.

### Skannaan nmapilla aiemmasta tehtävästä tuttua IP-osoitetta.
```
nmap -sn 192.168.56.103
```
parametri "-sn" ottaa portti skannauksen pois käytöstä. [nmap manual, host discovery](https://nmap.org/book/man-host-discovery.html)

![nmap metasploitable]()

### Tarkistetaan selaimessa, että meillä on oikea IP
Avaan Firefoxin Kalilla ja laitan URL-osoitteeksi:
```
http://192.168.56.103
```

### Jälkipohdinta tehtävästä
Olisikohan ideana ollut selvittää miten nmap olisi voinut löytää samasta verkosta Metasploitablen ilman tarkkaa osoitetta. Tulipahan mietittyä siihenkin ratkaisu. 
Voidaan ajaa nmap CIDR-verkolla ja se skannaa kaikki siinä verkossa olevat koneet. [nmap manual, target specification](https://nmap.org/book/man-target-specification.html)
```
nmap -sn 192.168.56.0/24
```
```
/24 määrittää verkko-osuuden IP-osoitteesta. Loput osoitteet skannataan.
```
Tällä komennolla meni muutama sekuntti kauemmin suorittaa skannaus. Tuloksetkin muuttuivat, nyt löytyi neljä  samassa verkossa.

![nmap_koko_verkko]()



## e) Porttiskannaa Metasploitable huolellisesti ja kaikki portit (nmap -A -T4 -p-). Poimi 2-3 hyökkääjälle kiinnostavinta porttia. 
Analysoi ja selitä tulokset näiden porttien osalta. Voit hakea analyysin tueksi tietoa verkosta, muista merkitä lähteet.

### Porttiskannaus
Komennolla:
```
nmap -A -T4 -p- 192.168.56.103
```
Tässä kesti muutama minuutti. Kun komento oli suoritettu näin todella monen portin olevan auki.

![Port scan 1]()

### Mitkä portit kiinnostaa

- 23 TELNET, salaamaton etähallinta. Tämä on vanhentunut ja haavoittuva protokolla.

- 80 HTTP, Verkkopalvelimien käyttämä. Useasti verkonkautta tapahtuvien hyökkäysten kohteena.
